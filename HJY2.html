프론트엔드 스쿨 4기 커리큘럼을 바탕으로
각 과정마다 어떤 방향으로 공부할지 학습 계획을 만들어보았습니다.

- Git 활용법 -

안드로이드 개발자로 일을 했을 때 Git을 처음 써봤는데
혼자 적응하려니까 뭐가 무슨 기능을 하는지 알기 어려웠습니다.
개발자라면 Git은 기본이라고 하는데 제대로 된 사용법을 모르니 답답했습니다.
인터넷에도 많이 서치 해봤지만 명확히 이해가 잘 되진 않더라고요.
그래서 전문가분들에게 배우면서 Git을 제대로 활용하는 법을 알아가고 싶습니다.

- HTML / CSS -

대학교 다닐 때 1~2년간 배웠었습니다.
하지만 졸업 한 지 2년이 되어가기도 하고 안드로이드 개발자로 일하면서
HTML, CSS를 쓸 일이 없어서 많이 잊어버렸습니다.
지금 이 홈페이지를 만들면서도 실력이 많이 떨어졌다고 느꼈습니다.
그래서 다시 기초부터 천천히 공부할 계획입니다.
이 부분은 좀 배웠던 부분이기 때문에 금방 따라갈 수 있을 거라고 생각합니다.

- RWD (반응형 웹) -

RWD은 학교에서 배우지 못해서 항상 하는 법이 궁금했습니다.
졸업 작품을 만들면서도 이 부분은 결국 해결하지 못했었습니다.
서치해 봐도 정확한 방법이 나오지 않아서 포기했던 때가 많았었습니다.
이걸 배우는 기회가 있으면 좋겠다고 생각했었는데 마침 커리큘럼에 있어서 좋다고 생각했습니다.
지금 홈페이지를 만들 때도 반응형 웹을 만들 줄 몰라서 생각대로 안 되었지만,
다음에 이걸 배우고 난 뒤에는 좀 더 완벽한 웹을 만들 수 있으면 좋을 것 같습니다.

- JavaScript -

자바스크립트는 아주 조금 배워서 거의 모르는 상태나 마찬가지입니다.
자바스크립트도 개발할 때 유용하게 쓰이는 걸로 알고 있어서 깊게 배워보고 싶었는데,
이 수업 들으면서 자세하게 알아가고 싶습니다.

- 중간 프로젝트 -

대학교 졸업작품을 했던 경험을 토대로 팀원들과 잘 협력하여 홈페이지를 만들고 싶습니다.
다른 사람과 협업하여 프로젝트를 한 경험이 많이 없기에 이 기회에
협업 능력을 기르고 싶습니다.
각자 더 자신 있는 부분을 맡아서 진행하면 더 순조롭고 좋게 결과가 나올 것 같습니다.

- React -

HTML이나 CSS는 배웠던걸 다시 배우는 거라면
React는 아예 모르는 걸 배우는 거다 보니 독학으론 힘들다고 생각합니다.
그래서 이 기회에 다른 사람들과 같이 배우면 실력도 금방 늘고 좋을 것 같다고 생각합니다.
React도 개발에 많이 쓰이는 걸로 알고 있어서 다른 것보다 좀 더 신경 써서 배워보고 싶습니다.

- 최종 프로젝트 -

여태 배운 내용과 만약 중간 프로젝트에서 피드백을 받았다면 그 피드백을 토대로
사람들과 협업하여 웹을 만들 자신이 있습니다.
마지막 프로젝트이다 보니 더 아이디어도 많이 내고 나온 아이디어들로 신중하게 구상하다 보면
좋은 홈페이지가 나올 거라고 생각합니다. 중간과 마찬가지로 각자 더 자신 있는 부분을 맡아서 진행하면 좋을 것 같습니다.
